1. Problem Statement

Structural engineers number columns manually in Revit — either floor by floor using tools like DiRoots, or entirely by hand. No existing tool handles cross-floor numbering consistently.

ProblemCurrent StateELLAH-ColNum Pro
Cross-floor numberingManual / per-viewOne click, all floors
Continuing existing marksMust track manuallyAuto-detects pattern
Grid-based naming (A1, B2)Not available in DiRootsBuilt-in
New columns on upper floorsMissed or renumberedDetected separately
Undo supportVaries by toolFull Ctrl+Z via Transaction

5. Core Logic — How It Works

Step 1 — Collect

RevitColumnCollector uses FilteredElementCollector(OST_StructuralColumns) to get all structural column elements. Reads: X/Y position, Base/Top Level, Family type, ColumnLocationMark (grid intersection A(64)-1(-46)), current Mark value.

Step 2 — Group

ColumnGrouper clusters columns by X,Y position within a tolerance (default 0.5 ft ≈ 15 cm). A column spanning floors 1–5 = one Revit element = one group. A new column starting on floor 3 = separate group.

Step 3 — Analyze Existing Marks

StatusMeaningAction
FullyNumberedAll floors have the same MarkKeep as-is
PartiallyNumberedSome floors have Mark, some emptyComplete to all floors
ConflictingDifferent floors have different MarksKeep most common, warn user
NotNumberedNo Mark anywhere in groupAssign next number

Step 4 — Assign (3 Modes)

ModeBehaviorUse Case
Smart ContinueDetects prefix+number pattern, keeps existing marks, continues from max+1Default — partial manual work exists
Add OnlyOnly numbers groups with zero marks. Skips everything elseConservative — do not touch anything existing
OverrideRenumbers everything from StartNumber. Existing marks replacedStart fresh — full renumber

Step 5 — Write

RevitMarkWriter writes ALL_MODEL_MARK parameter inside a named Revit Transaction. Full Ctrl+Z undo support. ReadOnly parameters are skipped safely. Returns count of updated elements.
